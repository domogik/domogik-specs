When a value is read and sent by 1 wire plugin, here is the rule which must be followed : .. code-block:

THEN use xpl-trig

DS18B20, DS18S20, DS2438¶

xPL messages¶

When temperature changes for a DS18B20 (or DS18S20) component : .. code-block:

xpl-stat
{
...
}
sensor.basic
{
device=
current=
type=temp
}

When temperature doesn’t change : .. code-block:

xpl-trig
{
...
}
sensor.basic
{
device=
current=
type=temp
}

For DS2438, for the humidity the same messages will be used with __type=humidity__
